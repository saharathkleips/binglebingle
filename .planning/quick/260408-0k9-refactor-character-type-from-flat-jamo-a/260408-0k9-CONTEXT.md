<decisions>
## Implementation Decisions

### Jamo type
- Introduce a `Jamo` type — the union of all valid Hangul Compatibility Jamo strings
  (consonants, double consonants, vowels, complex vowels, compound batchim)
- Slots in `Character` use `Jamo`, not raw `string`, for compile-time safety
- Derived from the existing keys in `jamo-data.ts` tables

### Character shape
- `Character = { choseong?: Jamo; jungseong?: Jamo; jongseong?: Jamo }`
- Empty character (`{}`) replaces length-0 array — no separate null/empty concept needed
- Pool items, guess slots, and target words all use this same type uniformly

### combine() rules (fully specified)
All rules operate on two single-position Character objects. The target state determines
how incoming jamo is interpreted:

**Empty target:**
- `+ { choseong }` → `{ choseong }`
- `+ { jungseong }` → `{ jungseong }` (valid intermediate; requires explicit ㅇ choseong to complete)

**Choseong-only target:**
- `+ { choseong }` → double consonant via `combineJamo` → `{ choseong: combined }`, or null if no rule
- `+ { jungseong }` → `{ choseong, jungseong }` (regardless of drag order — consonant always becomes choseong)

**Jungseong-only target:**
- `+ { jungseong }` → complex vowel via `combineJamo` → `{ jungseong: combined }`, or null if no rule
- `+ { choseong }` → `{ choseong, jungseong }` (same consonant-becomes-choseong rule)

**Choseong + jungseong target:**
- `+ { jungseong }` → apply vowel combination to existing jungseong via `combineJamo`, or null
- `+ { choseong }` → incoming choseong is treated as jongseong → `{ choseong, jungseong, jongseong: incoming.choseong }`

**Choseong + jungseong + jongseong target:**
- `+ { choseong }` → compound batchim via `upgradeJongseong`, or null
- `+ { jungseong }` → null (invalid — no 4-part syllables in Korean)
- `+ { jongseong }` → null (invalid)

**Double consonant defaults to choseong:**
- `combine({ choseong: 'ㄱ' }, { choseong: 'ㄱ' })` → `{ choseong: 'ㄲ' }`
- When a choseong-only character is combined into a choseong+jungseong target, the incoming
  choseong (which may itself be a double consonant like ㄲ) is placed as jongseong

### resolveCharacter simplification
- No choseong and no jungseong → `null`
- Choseong only → return `choseong` (bare consonant jamo string)
- Jungseong only → return `jungseong` (bare vowel — reachable but not a complete syllable)
- Choseong + jungseong [+ optional jongseong] → `composeSyllable(choseong, jungseong, jongseong)`

### isComplete
- Requires at minimum choseong + jungseong to produce a syllable block
- ㅇ is treated as a regular consonant (no special-casing for "silent" placeholder)
- Same codepoint range check: U+AC00–U+D7A3

### Decomposition
- Syllable (all 3 filled) → removes jongseong → `{ choseong, jungseong }`
- Then → removes jungseong → `{ choseong }`
- Compound batchim jongseong decomposes into its two constituent consonants (separate Characters)
- Keeps `decomposeSyllable` in composition.ts as the underlying utility; adds a `decompose(char)` in character.ts

### Ordering is not a concern
- `combine` is responsible for placing jamo in the correct slot regardless of drag direction
- No caller needs to pre-sort or pre-normalize inputs

### What does NOT change
- `combineJamo`, `upgradeJongseong`, `composeSyllable`, `decomposeSyllable` in `composition.ts` — unchanged
- `rotation.ts` and `jamo-data.ts` — unchanged
- The `character/` module boundary — `src/lib/` stays React-free

### Claude's Discretion
- Exact `Jamo` type implementation (string literal union vs branded string) — choose the approach
  that TypeScript can validate without excessive boilerplate; a union of all valid jamo codepoints
  is likely best
- File layout within `character/` — add `jamo-type.ts` or extend `types.ts` as fits best
- Whether `decompose` is exported or kept internal — expose if the reducer will need it

</decisions>
