# Phase quick-260408-0k9 Plan 01: Refactor Character Type Summary

**One-liner:** Keyed `{ choseong?, jungseong?, jongseong?: Jamo }` Character shape with a `combine()` state-machine replacing the flat `{ jamo: string[] }` array.

## What Was Built

Replaced the index-based `{ jamo: readonly string[] }` Character type with a semantically named, compile-time-safe keyed shape. Introduced a `Jamo` string-literal union covering all 45 valid Hangul Compatibility Jamo (basic consonants, double consonants, compound batchim, basic vowels, complex vowels). Added `combine()` as a deterministic state-machine that routes incoming jamo to the correct slot regardless of drag order.

## Implementation Notes

### types.ts
- `Jamo` union covers 45 jamo: 14 basic consonants + 5 double consonants + 11 compound batchim + 10 basic vowels + 11 complex vowels (some jamo appear in multiple tables — deduplicated in the union).
- `Character` uses three optional `Jamo` slots — empty object `{}` is the canonical empty state.

### character.ts
- `combine(a, b)`: if/else branches on which slots `a` has filled (empty / choseong-only / jungseong-only / choseong+jungseong / full). Calls `combineJamo`, `upgradeJongseong` from composition.ts with `as Jamo` cast (values are always valid by construction).
- `resolveCharacter`: reads slots directly — zero index arithmetic. Falls through to `composeSyllable` when both choseong and jungseong are present.
- `isComplete`: unchanged U+AC00–U+D7A3 range check on the resolved codepoint.
- `decompose`: builds `JONGSEONG_SPLIT_MAP` (reverse of `JONGSEONG_UPGRADE_RULES`) at module load. Splits compound batchim into two `{ choseong }` Characters when present.
