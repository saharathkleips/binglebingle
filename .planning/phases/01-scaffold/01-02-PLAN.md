  <action>
Create `.oxlintrc.json` at the repo root. oxlint uses a JSON config file. Enable the recommended rule groups for a strict TypeScript React project:

```json
{
  "$schema": "https://raw.githubusercontent.com/oxc-project/oxc/main/crates/oxc_linter/src/config_schema.json",
  "env": {
    "browser": true,
    "es2022": true
  },
  "globals": {
    "describe": "readonly",
    "it": "readonly",
    "test": "readonly",
    "expect": "readonly",
    "beforeEach": "readonly",
    "afterEach": "readonly",
    "beforeAll": "readonly",
    "afterAll": "readonly"
  },
  "rules": {
    "no-unused-vars": "error",
    "no-console": "warn",
    "no-debugger": "error"
  },
  "plugins": ["react", "typescript", "unicorn"],
  "categories": {
    "correctness": "error"
  }
}
```

IMPORTANT: The exact schema URL and available plugins may differ between oxlint versions. After writing this config, run `pnpm lint` to check for any config parse errors. If the `$schema` URL causes an error or the `plugins` field syntax is wrong for the installed oxlint version, check `pnpm exec oxlint --help` for the correct config format, then adjust.

The key requirements are:
1. `"browser": true` in env (we are a browser app)
2. `"es2022": true` in env (modern JS)
3. Vitest globals declared so oxlint does not flag `describe`, `it`, `expect` as undefined
4. At minimum `"correctness"` category enabled as errors

After creating the config, run:
```bash
pnpm lint
```

If there are lint violations in the scaffolded src/ files, fix them. Common issues to fix:
- Unused imports → remove them
- `console.log` statements → remove them (the scaffold should have none)
- Any variable declared but not used → remove or use

The goal is `pnpm lint` exits with code 0 and zero error-level violations.

  <action>
Run oxfmt on the src/ directory to auto-format all files:

```bash
pnpm fmt
```

This runs `oxfmt src/` which will reformat any files that do not match oxfmt's style. oxfmt is a Prettier-compatible formatter — it handles TypeScript, TSX, and CSS files.

After formatting, run the check command to verify no formatting violations remain:

```bash
pnpm fmt:check
```

This runs `oxfmt --check src/` and should exit 0.

If `pnpm fmt:check` still fails after running `pnpm fmt`, investigate which files are failing:
```bash
pnpm exec oxfmt --check src/ 2>&1
```

Common oxfmt issues:
- Trailing newline missing at end of file → oxfmt adds it automatically when run with `pnpm fmt`
- Quote style differences (single vs double) → oxfmt normalizes these
- Semicolon style → oxfmt enforces based on its defaults

After `pnpm fmt:check` passes, also confirm `pnpm fmt:check` specifically targets `src/` (not the whole repo — config files, playwright.config.ts, etc. can be formatted separately in a later phase). The scripts in package.json are: `"fmt": "oxfmt src/"` and `"fmt:check": "oxfmt --check src/"`.

If the package.json scripts do not yet have the correct targets (they may have been created in plan 01), verify and update them:
- `"fmt"` should be `"oxfmt src/"` (not just `"oxfmt"`)
- `"fmt:check"` should be `"oxfmt --check src/"`
