# Milestone: Design Tokens

**Status:** Pending
**Affected Slices:** `src/index.css`

## Context

빙글빙글 is a single-player Korean word-guessing game where players manipulate jamo (Korean alphabet components) on tile-like cards. The game currently has functional gameplay but no visual identity — tiles are plain white 3rem squares with gray borders, layout is a generic centered column, and there is only one animation (a CSS shake on rejected compose).

Milestone 1.4.x (단청/Dancheong) establishes a distinctly Korean visual identity rooted in 단청 (traditional decorative painting), 오방색 (five cardinal colors), and 화투 (hwatu flower cards). This first sub-milestone lays the CSS custom property foundation that every subsequent visual change builds on.

**Stack:** TypeScript strict + React 19 + CSS Modules + native CSS + Vite. pnpm only. No alternative styling solutions.

**Current token state:** `src/index.css` defines ~12 CSS custom properties under `:root` — a single `--size-tile: 3rem`, basic spacing/radius, generic gray/white/blue colors, and Wordle-style green/yellow/gray for game state.

## Goal

Replace the generic color and sizing token system with a Korean-traditional palette and hwatu-inspired tile dimensions, establishing the visual foundation that all subsequent 1.4.x milestones build on.

## Design Notes

**Obangsaek (오방색)** is the Korean traditional five-color system representing cardinal directions and elements. Two palette registers are provided: a vivid set for bold accents (borders, highlights, game state) and a pastel set for surfaces and backgrounds. The palettes are not locked — additional colors may be added if the design calls for them. The important thing is that both registers exist as named tokens.

**Tile ratio** is 3:5 (width:height), inspired by hwatu card proportions which are narrower than western playing cards.

**Semantic tokens** are critical for extensibility — the designer wants to be able to make small tweaks and micro-adjustments easily. Every visual decision should flow through a semantic token, not reference the palette directly in component CSS.
