# Milestone: GSAP Drag System

**Status:** Pending
**Affected Slices:** `src/lib/animation/` (new), `src/components/pool/`, `src/components/submission-area/`

## Context

빙글빙글 currently implements drag-and-drop using raw pointer events with manual ghost element creation. Both `Tile.tsx` (~100 lines of drag code) and `SubmissionSlot.tsx` (~80 lines) implement nearly identical systems: pointer capture, 4px drag threshold, DOM ghost element creation/positioning/cleanup, `elementsFromPoint` hit testing, and `data-drag-over` attribute management.

This milestone replaces that entire system with GSAP Draggable, which handles touch normalization, bounds clamping, click-vs-drag differentiation, and smooth motion out of the box. The tile itself moves during drag — no ghost elements.

**Prerequisites:**

- 1.4.1 (Design Tokens) — palette and tile dimension tokens
- 1.4.2 (Tile Visuals) — tiles have their final hwatu card appearance
- 1.4.3 (Game Layout) — pool canvas container exists with stable bounds for Draggable

**Current drag interactions to preserve:**
| Operation | Source → Target | Reducer Action |
|-----------|----------------|----------------|
| Compose | Pool tile → Pool tile | `CHARACTER_COMPOSE` |
| Insert | Pool tile → Submission slot | `SUBMISSION_SLOT_INSERT` |
| Swap | Slot tile → Slot tile | `SUBMISSION_SLOT_MOVE` |
| Rotate | Tap pool tile (rotatable) | `CHARACTER_ROTATE_NEXT` |
| Decompose | Tap pool tile (decomposable) | `CHARACTER_DECOMPOSE` |
| Remove | Tap filled slot | `SUBMISSION_SLOT_REMOVE` |

**Stack:** TypeScript strict + React 19 + CSS Modules + Vite. pnpm only.

**Environment warning:** The devcontainer has an outbound firewall — `pnpm add gsap` may be blocked. If package installation fails, **stop and ask** per CLAUDE.md. Do not attempt workarounds.

## Goal

Install GSAP, create a typed animation utility layer, and replace the manual pointer-event drag code with GSAP Draggable. Tiles move directly (no ghost elements). Core drag animations — pick up, put down, and nearby repositioning — are included.

## Design Notes

**No ghost elements.** The current system creates a detached `<div>` ghost that follows the pointer while the original tile dims. With GSAP Draggable, the tile element itself moves. This is simpler, more performant, and enables pick-up/put-down animations on the actual element.

**Reposition behavior (not "snap back").** When a drag ends without a valid drop target, or when a compose attempt is rejected, the tile should NOT fly all the way back to its starting position. Instead, it should animate to a nearby position where it doesn't overlap other tiles. Think of it like placing a card down on a table near where you tried to put it. This reposition logic (finding a nearby non-overlapping position) is very similar to what decomposition will need in 1.4.5 (splitting one tile into two that don't overlap). Consider designing the `animateReposition` helper to be reusable for both cases.

**Slot-to-slot swap animation.** When a filled slot tile is dragged to another filled slot, both tiles swap. This needs visual treatment — both tiles should animate to their new positions. The exact feel is TBD (simultaneous slide, slight arc, brief lift) but it should feel physical and decisive, like sliding cards across a table.

**Drop target detection** continues using `document.elementsFromPoint()` with `data-tile-id` and `data-slot-index` attributes. This is orthogonal to how the tile moves visually and works with Draggable's `onDragEnd` callback.

**GSAP Draggable** handles touch normalization, bounds clamping, and click-vs-drag differentiation out of the box. This eliminates the need for the manual 4px drag threshold, pointer capture, and cleanup logic.

**Test considerations:** GSAP Draggable relies on DOM measurement (`getBoundingClientRect`, element dimensions) which jsdom doesn't support well. Component tests for drag behavior may need to become E2E tests, or GSAP can be mocked in unit tests. The animation helper functions themselves can be unit-tested by asserting they return GSAP Tween/Timeline objects.
