# Milestone: Tile Visuals

**Status:** Complete
**Affected Slices:** `src/components/pool/`, `src/components/history-area/`, `src/components/submission-area/`

## Context

빙글빙글 tiles are currently plain white 3rem squares with 1px gray borders. This milestone transforms them into hwatu-style cards with a distinctly Korean aesthetic.

**Prerequisite:** 1.4.1 (Design Tokens) must be complete — this milestone consumes the obangsaek palette and tile dimension tokens defined there.

**Current tile components:**

- `Tile.tsx` / `Tile.module.css` — interactive pool tiles (drag, tap to rotate/decompose)
- `HistoryTile.tsx` / `HistoryTile.module.css` — read-only tiles showing guess results (correct/present/absent)
- `SubmissionSlot.tsx` / `SubmissionSlot.module.css` — slots that can be empty or filled with a tile

**Current drag system:** Both `Tile.tsx` and `SubmissionSlot.tsx` create manual DOM "ghost" elements during drag (a `<div>` appended to `document.body` with inline styles). These ghost elements will be **removed entirely** in milestone 1.4.4 when GSAP Draggable replaces the pointer-event system. In this milestone, the ghost styles should be updated to roughly match the new tile dimensions so drag doesn't look broken, but don't invest heavily in perfecting them — they're temporary.

**Stack:** CSS Modules + native CSS. No alternative styling solutions.

## Goal

All tile-like elements adopt the hwatu card aesthetic: 3:5 portrait ratio, thick red border treatment, warm color scheme, and character-only face.

## Design Notes

**Hwatu (화투) cards** have a distinctive thick red border and are narrower than western playing cards. The tile face should be clean — character only, no decorative motifs (deferred to a future milestone pending design exploration).

**Card sizing on mobile** is important — the tiles need to feel good to tap and drag on a small phone screen. The 3:5 ratio is set, but the absolute size (`--size-tile-width` / `--size-tile-height`) may need tuning once we see them on device. The token system from 1.4.1 makes this a single-line change.

**Hover states and glow effects** can be explored here if they feel natural — e.g., a subtle glow on tappable tiles, a border color shift on hover. These are low-cost CSS additions that add polish.
