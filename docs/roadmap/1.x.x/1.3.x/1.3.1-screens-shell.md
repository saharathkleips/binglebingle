# Milestone: Screens and Shell

**Status:** Pending
**Requirements:** UI-05, UI-06, UI-07, UI-08

## Goal

The complete game shell assembled — NavBar, How to Play screen, Results modal, and dev settings panel — making the product navigable and self-explanatory for a new player.

## Requirements

- [ ] **UI-05**: Game shell renders NavBar and routes between game area and instructions screen
- [ ] **UI-06**: How to Play screen explains the rotate → combine → compose mechanic to new players
- [ ] **UI-07**: Results modal appears on win/loss; shows score and guess count summary
- [ ] **UI-08**: Dev settings panel (visible in dev mode only) allows overriding word selection strategy (daily / random / fixed word)

## Success Criteria

1. NavBar renders on every screen and routes between the game area and the How to Play screen
2. How to Play screen explains the rotate, combine, and compose mechanic clearly enough for a new player
3. Results modal appears on win, showing the player's score and guess count
4. Dev settings panel is visible in development mode and allows switching word selection strategy; absent in production builds
