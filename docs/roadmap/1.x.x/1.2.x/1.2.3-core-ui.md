# Milestone: Core UI

**Status:** Pending
**Affected Slices:** `src/components/rack/`, `src/components/composer/`, `src/components/board/`

## Goal

Rack, Composer, and Board components rendered in the browser; a player can rotate jamo tokens, drag tokens into the Composer, compose a syllable, place it into a submission slot, and submit a guess.

## Execution Ledger

_(Agent: update these checkboxes as tests pass and local commits are made)_

- [ ] **UI-01** (`src/components/rack/`): Rack displays the jamo pool; player can tap a token to rotate it to the next jamo in its rotation set
- [ ] **UI-02** (`src/components/composer/`): Composer allows player to assemble jamo into a syllable, combine jamo, and decompose composed jamo back into constituents
- [ ] **UI-03** (`src/components/board/`): Board displays guess history as a grid; each evaluated tile is colored by result (correct / present / absent)
- [ ] **UI-04** (`src/components/rack/`, `src/components/composer/`): Player can drag jamo tokens from Rack to Composer (pointer and touch sensors via @dnd-kit)

## Success Criteria

1. Player can tap a token in the Rack to cycle it to the next jamo in its rotation set
2. Player can drag a jamo token from the Rack to the Composer
3. Player can assemble jamo in the Composer and see the resolved character update in real time
4. Player can place a completed character into a submission slot and submit the guess
5. Submitted guess rows appear on the Board with tiles colored by evaluation result
