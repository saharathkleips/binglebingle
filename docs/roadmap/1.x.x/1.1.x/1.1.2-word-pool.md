# Milestone: Word and Pool

**Status:** Complete
**Affected Slices:** `src/lib/word/`, `src/lib/puzzle/`, `src/lib/character/`, `src/lib/jamo/`

## Goal

The `Word` type, pool derivation pipeline, and word loading/selection implemented and tested.
Implementation diverged from the original spec: `Word` is `readonly CompleteCharacter[]` (not a branded string), pool derivation and word selection were extracted to `src/lib/puzzle/`, and normalization helpers were added to `character/` and `jamo/`.

## Execution Ledger

_(Agent: update these checkboxes as tests pass and local commits are made)_

- [x] **WORD-01** (`src/lib/word/`): `Word = readonly CompleteCharacter[]`; `createWord()` validates a raw string and returns a `Word` or `null`; `wordToString()` converts back to a Unicode string
- [x] **WORD-02** (`src/lib/puzzle/`): `fullDecompose()` recursively decomposes a Word to basic single-jamo Characters; `normalizeCharacter()` in `character/` rotates each pool element to the 0-index of its rotation set (obscuring the target word); `normalizeJamo()` added to `jamo/rotation` as the underlying helper
- [x] **WORD-03** (`src/lib/puzzle/`): `loadWords()` fetches `public/data/words.json`; `selectWord()` selects a word by strategy (`daily`, `random`, `fixed`, `byDate`); `WordSelectionStrategy` discriminated union defined here

## Success Criteria

1. `createWord()` rejects strings containing non-syllable characters and accepts valid words like '한국어'
2. `fullDecompose(word).map(normalizeCharacter)` produces the correct flat normalized jamo pool (e.g. ㄴ→ㄱ, ㅓ→ㅏ)
3. `loadWords()` fetches and parses words.json without error (tested with a fixture)
4. `selectWord()` with strategy `daily` returns the same word for the same date; with `random` it returns a valid `Word`
5. All exported functions have colocated Vitest tests that pass
